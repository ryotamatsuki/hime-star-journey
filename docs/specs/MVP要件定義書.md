# MVP要件定義書

## 目的

本書は「ひめの小さな星めぐり（仮）」MVPの実装要件を定義します。MVP範囲は、松山市内の道後温泉エリアと松山城エリアに限定します。

## 技術要件

- Unity、Godot、Unreal、Phaser、PixiJS、Kaboom.js、その他ゲームエンジン／ゲームフレームワークを使用しない。
- TypeScript、Vite、HTML Canvas 2D API、DOM UI、CSS、localStorageを使用する。
- 描画、UI、ロジック、データ、セーブ、アニメーション処理を分離する。
- 画像読み込みに失敗してもCanvas代替図形で進行できる。

## MVPに含めるもの

- タイトル画面
- プロローグ
- 星地図
- 道後温泉探索
- 松山城探索
- 会話UI
- 敵シンボル
- 通常敵8種
- 敵シンボル14体前後
- 敵1体または2体のエンカウント
- カード式ターン制バトル
- ターゲット選択UI
- カゲマサ戦
- 旅の手帳10項目前後
- オートセーブ
- つづきから
- 手描き絵本風アセット
- 2.5D風アニメーション

## ストーリー要件

MVPでは、旧企画書 v0.7 のストーリーを正本として扱い、以下をゲーム内の導入、会話、進行目標、星地図、エンディングで伝える。

- ひめは家族旅行で道後温泉へ向かう。
- 出発前に、おばあちゃんから小さなみかん色の星がついたペンダントを受け取る。
- ひめの家は、かつて土地の星を見守った「星守り」の家系につながっている。ただし、家族もおばあちゃんも詳しい役目は知らない。
- ペンダントは白鷺のお守りであり、愛媛各地の「土地の星」を映す星地図の器である。
- 道後温泉の夜、ひめはペンダントに宿っていた守り鳥シロと出会う。
- カゲマサの手下が、ペンダント中央の「みかん星の核」を奪う。
- ペンダントの外枠と鎖はひめの手元に残り、弱い星地図機能と星のカードは使える。
- みかん星の核を奪われたことで、道後の湯がぬるくなり、提灯が青く光り、湯けむりがざわつき、松山城の方角に黒い影がのびる。
- ひめの動機は、ペンダントの光を取り返す、シロを助ける、道後温泉と松山城を元に戻す、星地図を完成させる、の4つである。
- カゲマサはMVPの章ボスであり、物語全体の最終黒幕ではない。
- 全体黒幕候補の「くろぼし」は、MVPでは完全には明かさず、次の冒険につながる謎として残す。
- MVPでは、道後の「湯の星」と松山城の「城の星」を取り戻す。
- MVP終了時、ペンダントの星地図に未解放の空白の星を表示する。

### フェーズ別実装範囲

P6では、画像生成とアニメーション付きプロローグ、道後温泉導入、道後温泉クリア、湯の星入手、松山城解放までを主対象とする。カゲマサ戦、みかん星の核の完全奪還、城の星入手、MVPエンディングはP7以降または後続フェーズで扱う。

## MVPに含めないもの

- オンライン機能
- ランダムエンカウント
- 1対3以上の戦闘
- 複数味方UI
- 仲間加入イベント
- 装備変更画面
- ショップ
- 経験値稼ぎ
- 広大なワールドマップ
- 正式な全アセット完成
- 愛媛県20市町すべての実装

## 機能要件

### タイトルとセーブ

- 新規開始ができる。
- セーブデータがある場合は「つづきから」が有効になる。
- localStorageにセーブする。
- 主要な進行更新時に自動保存する。

### プロローグ

- 家族旅行で道後温泉へ向かう導入を表示する。
- おばあちゃんが、ひめにみかん星のペンダントを渡す。
- ペンダントが白鷺のお守りであり、土地の星を映す星地図の器であることを、短い会話と演出で示す。
- 星守りの家系設定は、P6では説明しすぎず、ペンダントが代々受け継がれた大切なお守りであることを優先して見せる。
- 道後温泉到着後、湯けむりの中でシロと出会う。
- カゲマサの手下が、ペンダント中央のみかん星の核を奪うイベントを入れる。
- みかん星の核を奪われた後、道後温泉の異変を示す。
- ペンダントの外枠と弱い星地図機能が残ることを示し、P6以降もカードと星地図を使える状態にする。
- P6では、この導入から道後温泉クエスト開始までを実装対象にする。

### 星地図

- 道後温泉と松山城の星を表示する。
- 初期状態では道後温泉が解放済み、松山城は未解放とする。
- 道後温泉クリア後、湯の星または湯の星のかけらを取得し、松山城を解放する。
- MVP終了時に未解放の星を表示し、今後の拡張余地を示す。

### 探索

- ひめを上下左右に移動できる。
- シロはひめに追従または演出用に表示する。
- シロは、みかん星の核を奪われて弱っている守り鳥として扱い、短いヒントで道後温泉探索を案内する。
- 敵シンボルに接触すると戦闘に入る。
- 一度しずめた敵は復活しない。
- 必須敵をしずめると進行フラグが更新される。
- 道後温泉クリア時に、松山城へ向かう理由を黒い影、みかん星の核の奪還、星地図解放と結びつける。

### バトル

- 味方側はMVPではひめ1人だが、内部的には `partyMembers: BattleActor[]` で管理する。
- 敵側は `enemies: BattleActor[]` で管理する。
- 敵シンボルは `enemyId` ではなく `encounterId` を持つ。
- `EncounterData` により、敵1体または2体を生成できる。
- MVPでは1対3以上の戦闘を実装しない。
- 攻撃カードで敵が複数いる場合はターゲット選択UIを出す。
- 回復・防御カードはひめ自身に自動適用する。
- 敵1体だけを倒しても、残りの敵がいれば戦闘を継続する。
- 敵全員をしずめると勝利する。
- ボス戦はカゲマサ1体とする。
- カゲマサ戦後、みかん星の核を取り戻し、ペンダントの光を回復する。ただし、この完全奪還イベントはP7以降または後続フェーズの対象とし、P6では未実装でもよい。

### 旅の手帳

- 解放済みの星、カード、地域メモ、あらすじを閲覧できる。
- 手帳を読まなくてもMVPは進行できる。

## データ要件

- `EnemySymbolData` は `encounterId` を持つ。
- `EncounterData` は `enemies: EncounterEnemyData[]` を持つ。
- `BattleState` は `partyMembers` と `enemies` を配列で持つ。
- `SaveData` は `partyMemberIds` と `activePartyMemberIds` を持つ。
- MVP初期値は `partyMemberIds: ["hime"]`、`activePartyMemberIds: ["hime"]` とする。

## アセット要件

- アセット生成プロンプトは `docs/asset-prompts/` に保存する。
- 生成画像は `public/assets/generated/` 以下に用途別に配置する。
- 主要ビジュアル参照は `docs/visual-reference/key-visuals/` に保持する。
- 背景、キャラ、敵、ボス、カード、UI、エフェクトのカテゴリを分ける。

## 受入条件

- タイトル画面から新規開始できる。
- つづきから再開できる。
- 道後温泉を探索できる。
- 松山城を探索できる。
- 星地図で松山城が解放される。
- 通常敵8種が登場する。
- 複数敵エンカウントが3件以上ある。
- 攻撃カード使用時、敵が複数いる場合はターゲット選択UIが表示される。
- 敵1体だけを倒した場合、残りの敵との戦闘が継続する。
- 敵全員をしずめると勝利する。
- カゲマサを星封じで再封印できる。
- MVPエンディングまで通して遊べる。
